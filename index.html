Formation › Phase 1 — Fondations
Phase 1 · Mois 1–3 · Fondations

Consolider les fondations

Passer de scripts qui fonctionnent à du code propre, versionné et compréhensible — la base incontournable pour travailler avec des développeurs.

⏱ ~12 semaines 2h / semaine ~24h au total 5 modules
Progression Phase 1 0 / 5 modules
M1
Python solide : structures, fonctions, OOP
Passer de "ça marche" à du code propre, lisible et maintenable que d'autres peuvent reprendre.
3 semaines Intermédiaire Fondation critique
Objectifs concrets
🏗️Comprendre les classes et l'OOP pour structurer son code
🧩Créer des fonctions réutilisables avec paramètres et retours
⚠️Gérer les erreurs proprement (try/except)
📦Comprendre les modules et organiser ses fichiers
🔁Réécrire un script existant de façon propre
Programme semaine par semaine
Sem 1
Fonctions & portée des variablesthéorie
Paramètres, valeurs par défaut, *args/**kwargs. Exercice : refactoriser une fonction longue en sous-fonctions.
Sem 2
Classes & POO basiquethéorie
Attributs, méthodes, héritage simple. Comprendre le concept pour lire le code des autres et utiliser des librairies.
Sem 3
Gestion d'erreurs + modules + projetpratique
try/except/finally, imports propres. Réécrire un de tes vrais scripts avec tout ça.
Ressources recommandées
🎓
Python for Everybody – Coursera (Univ. Michigan)
Le plus pédagogique. Modules 3 à 5 pour ta cible. Audit gratuit disponible.
Gratuit
📖
Real Python – realpython.com
Articles ultra-pratiques. Cherche "OOP Python", "Python functions", "error handling".
Gratuit
🏋️
Exercism.io – Python track
Exercices progressifs avec correction. Faire 2-3 exercices par semaine pour ancrer les concepts.
Gratuit
🛠 Mini-projet
Nettoyer et restructurer un vrai script
Prends un script Python que tu as déjà et réécris-le proprement. Quelqu'un d'autre doit pouvoir le lire sans te demander d'explications.
01Découpe en fonctions logiques (une fonction = une action)
02Ajoute des docstrings à chaque fonction
03Ajoute une gestion d'erreur sur les parties qui peuvent planter
04Crée un README.md qui explique à quoi sert le script
💡 Comment utiliser l'IA ici : Au lieu de demander "écris-moi ce script", demande "explique-moi pourquoi tu as utilisé une classe ici plutôt qu'une fonction". L'IA devient un prof patient plutôt qu'un générateur de code opaque.

M2
Git & GitHub : versionner son code
L'outil le plus important pour travailler avec des développeurs. Pas optionnel.
1 semaine Débutant Priorité absolue
Objectifs concrets
💾Initialiser un repo, faire des commits clairs
🌿Créer et merger des branches
🤝Comprendre les Pull Requests
📤Pousser son code sur GitHub
Programme en 1 semaine
J 1–2
Concepts de basethéorie
init, add, commit, push, pull. Créer ton premier repo GitHub.
J 3–4
Branches & mergepratique
Créer une branche, faire des modifications, merger. Gérer un conflit simple.
J 5
Mettre tes scripts sur GitHubprojet
Crée un repo "mes-automatisations" et versionne tes scripts existants avec de bons README.
Ressources recommandées
🎮
Oh My Git! – ohmygit.org
Un jeu qui enseigne Git visuellement. La façon la plus rapide de comprendre les concepts.
Gratuit
📖
Git Handbook – GitHub Docs
La référence officielle. Lire la section "Get started" en entier.
Gratuit
🎥
"Git and GitHub for Beginners" – freeCodeCamp YouTube
Tutoriel vidéo d'1h qui couvre tout l'essentiel. Idéal pour voir avant de faire.
Gratuit
💡 Pourquoi c'est non-négociable : Quand tu interagiras avec des devs, tout passe par Git. Un chef d'équipe qui ne comprend pas Git ne peut pas relire les contributions de son équipe, comprendre l'historique, ou participer à une code review.

M3
SQL avancé : requêtes complexes & agrégations
Tu nettoies déjà de la donnée — le SQL avancé te permet de le faire à l'échelle et de dialoguer avec les data engineers.
3 semaines Intermédiaire Compétence différenciante
Objectifs concrets
🔗Maîtriser les JOIN (INNER, LEFT, RIGHT, FULL)
📊GROUP BY, HAVING, fonctions d'agrégation
🪟Window functions basiques (RANK, LAG, LEAD)
🔄Sous-requêtes et CTEs (WITH...AS)
⚡Notions de performance et optimisation
Programme semaine par semaine
Sem 1
Jointures maîtrisées + agrégationsthéorie
Les JOIN sont au cœur de 80% des requêtes complexes. Exercices : croiser des tables clients, commandes, produits.
Sem 2
Sous-requêtes, CTEs, CASE WHENthéorie
CTEs (WITH...AS) pour rendre le code lisible. CASE WHEN pour créer des colonnes calculées.
Sem 3
Window functions + projet réelprojet
RANK, ROW_NUMBER, LAG/LEAD. Puis : prendre un vrai jeu de données marketing et répondre à 5 questions business en SQL pur.
Ressources recommandées
📊
Mode Analytics SQL Tutorial
Meilleure ressource gratuite pour le SQL analytique. Très axée cas concrets.
Gratuit
🧩
SQLZoo – sqlzoo.net
Exercices interactifs progressifs. Faire les sections JOIN, SUM and COUNT, More JOIN.
Gratuit
🏆
StrataScratch – stratascratch.com
Questions SQL réelles posées en entretien chez des boîtes tech. Excellent pour tester son niveau.
Gratuit (tier limité)
💡 Utilisation IA : Soumets tes requêtes SQL à Claude et demande "optimise cette requête et explique pourquoi ta version est meilleure". C'est l'un des meilleurs cas d'usage : tu vois du code, tu comprends le raisonnement.

M4
Bases du web : HTML / CSS / JS pour non-développeurs
Pas pour coder des sites. Pour comprendre comment une page web fonctionne et parler aux devs frontend avec pertinence.
3 semaines Débutant Culture technique
Objectifs concrets
🌐Comprendre la structure HTML d'une page
🎨Lire et modifier du CSS sans paniquer
⚡Comprendre ce que fait JS (sans l'écrire)
🔍Utiliser l'inspecteur du navigateur (F12)
💬Parler de DOM, events, responsive avec des devs
Programme semaine par semaine
Sem 1
HTML : la structurethéorie
Balises, attributs, DOM. Ouvrir l'inspecteur sur un site du travail et comprendre l'arbre HTML.
Sem 2
CSS : la mise en formethéorie
Sélecteurs, box model, flexbox. Modifier le style d'un composant sans casser le reste.
Sem 3
JavaScript : comprendre la logiquepratique
Variables, fonctions, events, fetch API. Comprendre "cet élément réagit à un clic et appelle une API".
Ressources recommandées
⚒️
The Odin Project – Foundations
Parcours complet HTML/CSS/JS. Très pédagogique. Faire les 4 premières sections.
Gratuit
📖
MDN Web Docs – Mozilla
LA référence. À utiliser comme dictionnaire quand tu tombes sur quelque chose d'inconnu.
Gratuit
🎮
CSS Flexbox Froggy + Grid Garden
Deux jeux pour comprendre flexbox et grid en 30 min. Vraiment efficace.
Gratuit
💡 Exercice concret : Prends une page web du travail, ouvre l'inspecteur (F12), identifie 3 composants et essaie de comprendre comment ils sont construits. Demande à l'IA d'expliquer les parties qui t'échappent.

M5
Projet pratique : automatiser un vrai flux de données
Synthèse de tout ce que tu as appris. Un projet concret, documenté et versionné sur GitHub.
2 semaines Intermédiaire Projet portfolio
Idées de projets (choisir 1)
📬Extraire des données d'une API (Mailchimp, HubSpot, GA) → nettoyer → pousser dans Google Sheets
📊Agréger des CSV avec Pandas et générer un rapport HTML automatique
🔔Script qui surveille un fichier, détecte des anomalies et envoie une alerte email
Livrables attendus
Code
Script Python propre, découpé en fonctions, avec gestion d'erreurs
GitHub
Repo public avec commits clairs et README expliquant le projet
Docs
Court document : problème résolu, architecture, limites et améliorations possibles
💡 Le vrai objectif : Ce n'est pas le projet en lui-même qui compte, c'est de montrer que tu sais penser, documenter et livrer. Un dev doit comprendre en 2 minutes ce que ça fait et pourquoi tu as fait ces choix — c'est exactement ce que tu attendras de ton équipe.
